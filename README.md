# Morris Worm Investigation

Safe KYPO/CyberRangeCZ-style training scenario inspired by the Morris Worm lab.

## Lab users created by provisioning

These are local training users inside the lab machines:

| Machine | Username | Password |
|---|---|---|
| attacker | attacker | attacker |
| analyst | analyst | analyst |
| victim1-3 | victim | victim |

KYPO sandbox SSH access can still be generated by the platform through the `user-access` role and pool credentials. The local users above are only for the scenario itself.

## Main files

- `topology.yml`: sandbox infrastructure
- `training.json`: levels/questions
- `provisioning/requirements.yml`: external CyberRangeCZ roles
- `provisioning/playbook.yml`: Ansible role execution
- `provisioning/roles/*`: local scenario roles

## Scenario

The attacker machine contains a safe simulator at `/home/attacker/morris/worm_sim.py`.
Victims listen on TCP port 9999 using a controlled service. When the simulator connects, victims create `/opt/morris/badfile` and append `/opt/morris/infection.log` for forensic analysis.
